Develop and offer content

Development of content and its offering goes beyond using, analysing and combining datasets and services available on the Urban TEP (U-TEP) platform. Here, the user can bring in additional expertise and functionality by providing a new “urban-related” dataset or data processor. The data set or processor might be developed in an external envi-ronment or generated on-site using the U-TEP platform. A data processor can be offered as a service, or it can be used to generate new datasets or systematically extend existing ones.

This service is open to well-known, registered users of the U-TEP. We hope for your understanding that during initial operations the registration of well-known users free-of-charge is restricted to a selected set of pilot organisations. To become a well-known user you further have to register with one of the U-TEP processing centres.

Developing your own data processors

Earth observation data processors are software elements that can be started and con-trolled by a processing system to generate an output data product from one or several input data products. With the concurrent or sequential application of a data processor to a larger input dataset an output dataset or value-added product can be produced.

Development, test, packaging, and upload of a processor to an U-TEP processing centre is supported by a set of tools. These tools are made available in a VM image that can be downloaded and used on your local machine. With few test data this allows for an efficient local test cycle. After upload the same processor that you have tested on EO data products on your local VM, the same functionality can then be applied to larger datasets in the cluster environment of a U-TEPprocessing centre. It is offered to you as an additional service in the U-TEP portal. You must own the data processor and the required software components or have the right to share them in order to upload a processor.

The data processor tested on the platform can be released for use by a specific community or any user. Publication of a result dataset is handled the same way as the publication of a user-provided dataset that has been uploaded. Systematic generation and extension with new data can be requested for datasets of common interest where systematic generation is more adequate than processing on demand.

Downloading U–TEP’s VM for processor development and deployment

Download the VM if you want to develop, deploy, and use a data processor for Earth Ob-servation data on an U-TEP processing centre. The VM contains tools to test, package and upload processors. The VM can be run in a hypervisor (e.g. Virtual Box from Oracle) on your local machine.

  • You can download the VM image urbantep-vm-1.0.tar.gz from an Urban TEP processing centre.
  • Unpack the download package to a local directory and start it in your hypervisor. It is a Linux VM that comes with a graphical user interface.
  • Log in to the VM as user urbantep with default password urbantep.

Development, test, packaging, and upload of a data processor

A thematic urban data processor is a software element that uses an input data item (product) and generates an output data product. This relation is not necessarily one-to-one, but many data processors are. You may consider this as an on-demand service. Processors must be executable in Linux. They can be implemented in compiler-based languages (C/C++, Fortran, ...), runtime system languages like Java or Python or R or many others, down to simple shell scripts. The only restriction is that no foreign or com-mercial license shall be required to run the data processor because it will be started concurrently on the cluster infrastructure of a processing centre. To develop a data processor

  • Follow the instructions and examples of the Urban TEP Software User Manual
  • You can also find a short set of instructions in README provided in the VM on the desktop after login.
  • Processors must be locally tested with example input data product before up-load.
  • There are tools for packaging and for upload of your processor in the VM. Packaging usually involves putting your processor into a Docker container but this is done behind the scenes. You may specify dependencies of your processor if it has specific needs. You have to specify identification and parameterisation information for your processor in an XML descriptor file. Examples are provided in the VM. Upload is also supported by a tool that transfers your processor package to a processing centre.
  • The upload to the processing centre requires the authentication as well-known user at the processing centre. Deployment and availability of the processor in the portal may either be automated or partially supported by an operational procedure of the respective processing centre.
  • Test of your processor deployed in a processing centre is done via the portal. At that stage usually only you can use your processor on the Urban TEP platform.

Publication of a data processor for use by the community

A useful data processor can be offered to a particular community or all users. The pro-cessing centre and the U-TEP portal will modify the access authorisation information in order to provide access to the processor.






Proposal of a data processor for systematic processing

If there is a common interest in the systematic generation and data-driven extension of an urban product then U-TEP may decide to set up a systematic ingestion and pro-cessing chain. The processor to generate products systematically may also involve a user-provided processor.

Setup of systematic generation is handled by an operational procedure that involves governance, and subsequently the configuration and integration of a chain starting at the ingestion of new data products and ending in daily new entries in a defined dataset collection. Please understand that the systematic generation of additional datasets may be limited to cases that cannot be easily generated on-demand and that are of high interest for a broad spectrum of urban user communities.
